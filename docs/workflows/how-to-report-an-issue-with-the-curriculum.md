# Reporting Curriculum Issues

Our curriculum has been created with the help of countless volunteers and past participants. As we often talk about, the world of tech moves swiftly. Something that may have worked great 6 months ago may now be outdated and not work anymore. We need your help to keep the resources current and in working order for the next group of participants as well as for your cohort-mates. When you find an issue please do the following steps:

- Talk about the issue in the participants channel. There will be a thread for each topic you will be going over. Put a note in the tread and ping the TA and TPM to get immediate clarification. You are probably not the only person with this issue, so posting in the Slack thread will help everyone!

DO NOT SUFFER IN SILENCE! There is a huge correlation between people who ask lots of questions and reach out when they need help and those that do really well in the program. We know it is super scary if you are not used to asking for help, this is a safe space and we are here to help, but we need to know there are issues.

- If the issue is something that can be resolved quickly, such as a missing or broken link, confusing wording, etc. TPM and TA can update and push changes immediately.
- Create a bug report using this form 👈

OR

- The best and most helpful thing to do is to create PR. This is especially helpful if you have found the solution to the problem. You do not need to fully fix the curriculum, but creating a PR that shows us exactly where the problem is and any details you have about what needs to be done is very very helpful! Follow this guide to creating a curriculum PR
- What is not helpful, and will not help us fix the problem is giving general statements like "The curriculum needs work," or "The curriculum this week was confusing." Please make sure you give specific and actionable information so that we can keep improving the curriculum!
